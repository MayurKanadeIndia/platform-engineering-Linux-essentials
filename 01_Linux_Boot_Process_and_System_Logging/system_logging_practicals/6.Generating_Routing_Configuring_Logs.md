## 🎯 Analogy Summary

### The Post Office Analogy
```
Letters arrive from everywhere   →  Log messages from apps/kernel
Post office sorting rules        →  rsyslog routing rules
  "All letters to Mumbai → Truck A"  →  "auth.* → /var/log/auth.log"
  "Urgent mail → Express courier"    →  "*.crit → remote server"
Postmaster can ADD new rules     →  You edit /etc/rsyslog.d/*.conf
Postmaster can CREATE new routes →  You create custom log files

This chapter teaches you to BE the postmaster.
```

---

## 💡 Core Concept

> The default logging config works fine out of the box.
> But in production, you CUSTOMIZE it:
>   - Route your app logs to their own dedicated file
>   - Send critical logs to a remote server
>   - Filter out noise you don't need
>   - Control how journald uses disk space
> This is what separates a junior admin from a senior one.

## 🗂️ Configuration Files — Complete Map

| File | Purpose | When to Edit |
|---|---|---|
| `/etc/rsyslog.conf` | Main rsyslog config | Rarely — use drop-ins |
| `/etc/rsyslog.d/*.conf` | Drop-in routing rules | Add YOUR rules here |
| `/etc/systemd/journald.conf` | journald settings | Disk limits, retention |
| `/etc/logrotate.conf` | Main rotation config | Global defaults |
| `/etc/logrotate.d/*.conf` | Per-app rotation rules | Add YOUR app rules here |

### Drop-in Directory Loading Order
```
/etc/rsyslog.d/
├── 20-ufw.conf          ← loads first (lower number)
├── 50-default.conf      ← Ubuntu defaults
├── 60-myapp.conf        ← YOUR rules (add here)
└── 99-remote.conf       ← remote forwarding (loads last)

Files load in alphabetical/numerical order.
Lower numbers = higher priority.
```

## ⚙️ rsyslog Rule Syntax — Full Reference

```bash
# Basic format:
facility.severity    action

# ACTIONS:
facility.severity    /var/log/file.log        # write to local file
facility.severity    @192.168.1.100           # send via UDP
facility.severity    @@192.168.1.100          # send via TCP (reliable)
facility.severity    |/dev/xconsole           # send to console
facility.severity    :omusrmsg:*              # send to all users terminals
facility.severity    ~                        # DISCARD (drop it!)

# WILDCARDS:
kern.*               /var/log/kern.log        # ALL severities
*.err                /var/log/errors.log      # ALL facilities, err+
*.*                  /var/log/syslog          # EVERYTHING

# NEGATION (exclude):
*.*;auth.none        /var/log/syslog          # everything EXCEPT auth
*.*;kern.none;auth.none  /var/log/syslog      # except kern AND auth

# MULTIPLE FACILITIES:
auth,authpriv.*      /var/log/auth.log        # auth AND authpriv

# REMOTE FORWARDING:
*.*     @192.168.1.100          # UDP, port 514 (default)
*.*     @@192.168.1.100:514     # TCP, port 514 (production use)
*.crit  @@192.168.1.100:514     # only critical+ to remote
```

## 🖥️ Commands — rsyslog Configuration

```bash
# ── Viewing Config ──────────────────────────────────────────
sudo cat /etc/rsyslog.conf                     # view main config
sudo cat /etc/rsyslog.d/50-default.conf        # view default rules
ls -la /etc/rsyslog.d/                         # list all rule files

# ── Verifying & Applying Config ─────────────────────────────
sudo rsyslogd -N1                              # verify syntax (dry run) ⭐
sudo systemctl restart rsyslog                 # apply config changes ⭐
sudo systemctl status rsyslog                  # check rsyslog health
sudo systemctl reload rsyslog                  # reload without restart

# ── Testing Custom Rules ─────────────────────────────────────
logger -p local0.info    "MyApp: started"      # test custom facility
logger -p local0.warning "MyApp: high load"
logger -p local0.err     "MyApp: db timeout"
cat /var/log/myapp.log                         # verify entries appeared
sudo tail -f /var/log/myapp.log                # watch live
```

## 🖥️ Commands — journald Configuration

```bash
# ── Viewing journald Config ──────────────────────────────────
cat /etc/systemd/journald.conf                 # view config
cat /etc/systemd/journald.conf | grep -v "^#" | grep -v "^$"  # active lines

# ── Disk Usage & Cleanup ─────────────────────────────────────
journalctl --disk-usage                        # current journal size ⭐
sudo journalctl --vacuum-size=200M             # reduce to 200MB ⭐
sudo journalctl --vacuum-time=7days            # delete older than 7 days ⭐
sudo journalctl --vacuum-time=2weeks           # delete older than 2 weeks
sudo journalctl --vacuum-files=10              # keep only 10 journal files

# ── Persistence ─────────────────────────────────────────────
sudo mkdir -p /var/log/journal                 # enable persistence
sudo systemd-tmpfiles --create --prefix /var/log/journal
sudo systemctl restart systemd-journald
journalctl --list-boots                        # verify boot history works

# ── Applying Changes ─────────────────────────────────────────
sudo systemctl restart systemd-journald        # apply journald.conf changes
sudo systemctl status systemd-journald         # verify health
```

## 🖥️ Commands — logrotate

```bash
# ── Viewing Configs ──────────────────────────────────────────
ls /etc/logrotate.d/                           # list all rotation configs
cat /etc/logrotate.conf                        # main config
cat /etc/logrotate.d/rsyslog                   # syslog rotation rules

# ── Testing & Running ────────────────────────────────────────
sudo logrotate -d /etc/logrotate.d/myapp       # DRY RUN (test) ⭐
sudo logrotate -f /etc/logrotate.d/myapp       # force rotate NOW
sudo logrotate -d /etc/logrotate.conf          # dry run ALL configs

# ── Checking Status ──────────────────────────────────────────
cat /var/lib/logrotate/status                  # rotation history
ls -lh /var/log/syslog*                        # see rotated files
```

## 📋 journald.conf — Key Settings Reference

```ini
[Journal]

# Storage location:
Storage=auto         # auto = persistent if /var/log/journal/ exists
Storage=persistent   # always write to disk ⭐
Storage=volatile     # RAM only (lost on reboot)
Storage=none         # disable journald storage

# Compression:
Compress=yes         # compress old journal files (default, save space)

# Disk limits:
SystemMaxUse=500M    # total journal max size ⭐
SystemKeepFree=1G    # always keep this much free on disk
SystemMaxFileSize=50M  # max size of one journal file

# Retention:
MaxRetentionSec=1month  # delete logs older than this ⭐
MaxRetentionSec=2week
MaxRetentionSec=7day

# Forward to rsyslog:
ForwardToSyslog=yes  # yes = journald feeds rsyslog (default)

# Rate limiting (prevent flooding):
RateLimitIntervalSec=30s  # time window
RateLimitBurst=10000      # max messages per window
```

## 📋 logrotate Config — Options Reference

```bash
/var/log/myapp.log {
    daily              # rotate: daily | weekly | monthly | yearly
    rotate 14          # keep N rotated files
    compress           # gzip compressed rotated files
    delaycompress      # don't compress most recent rotated file
    missingok          # don't error if log file is missing
    notifempty         # don't rotate if file is empty
    create 640 syslog adm  # permissions for new log file
    dateext            # use date in filename (myapp-20260307.log)
    size 100M          # rotate when file reaches this size (alternative)
    maxage 30          # delete rotated files older than 30 days
    sharedscripts      # run postrotate once even for multiple files
    postrotate
        /usr/lib/rsyslog/rsyslog-rotate   # tell rsyslog to reopen files
    endscript
}
```

## 🏗️ Full Custom App Setup — Step by Step

### Complete Setup for a Custom "webstore" Application
```bash
# ── Step 1: Create log directory ────────────────────────────
sudo mkdir -p /var/log/webstore
sudo chown syslog:adm /var/log/webstore
sudo chmod 755 /var/log/webstore
sudo touch /var/log/webstore/app.log
sudo touch /var/log/webstore/error.log
sudo chown syslog:adm /var/log/webstore/*.log
sudo chmod 640 /var/log/webstore/*.log

# ── Step 2: Create rsyslog routing rule ─────────────────────
# File: /etc/rsyslog.d/60-webstore.conf
# local1.*     /var/log/webstore/app.log
# local1.err   /var/log/webstore/error.log
# local1.crit  @@192.168.1.100:514
# local1.*     ~

# ── Step 3: Create logrotate config ─────────────────────────
# File: /etc/logrotate.d/webstore
# /var/log/webstore/*.log { weekly; rotate 30; compress; ... }

# ── Step 4: Verify and apply ─────────────────────────────────
sudo rsyslogd -N1                    # check syntax
sudo systemctl restart rsyslog       # apply

# ── Step 5: Test ─────────────────────────────────────────────
logger -p local1.info    "Webstore: Application started"
logger -p local1.warning "Webstore: High cart abandonment"
logger -p local1.err     "Webstore: Payment gateway timeout"
logger -p local1.crit    "Webstore: Database pool exhausted"

# ── Step 6: Verify routing ───────────────────────────────────
cat /var/log/webstore/app.log        # all entries
cat /var/log/webstore/error.log      # only err + crit
sudo logrotate -d /etc/logrotate.d/webstore   # test rotation
```

## ✅ Key Takeaways

```
✅ Add rules to /etc/rsyslog.d/ — never edit main rsyslog.conf directly
✅ Rule format: facility.severity → destination
✅ Use ~ action to DISCARD noisy logs
✅ Use local0–local7 for YOUR custom application logs
✅ Always verify rsyslog config with rsyslogd -N1 before restarting
✅ @ = UDP forwarding, @@ = TCP forwarding (use TCP in production)
✅ journald Storage=persistent + /var/log/journal = logs survive reboots
✅ Use journalctl --vacuum to reclaim disk space
✅ Create a logrotate config for EVERY custom log file you create
✅ Test logrotate with -d (dry run) before forcing rotation
```
