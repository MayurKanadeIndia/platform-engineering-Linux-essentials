## рџ“„ Key Log Files вЂ” Detailed Reference

### 1. syslog / messages вЂ” The Main Diary
```
Location (Debian/Ubuntu): /var/log/syslog
Location (RedHat/CentOS):  /var/log/messages
Purpose: General system events вЂ” the first place to look for any problem
```
```bash
# Sample content:
Mar 07 10:00:01 myserver systemd[1]: Started Session 5 of user rahul.
Mar 07 10:05:01 myserver CRON[1234]: (root) CMD (/usr/bin/backup.sh)
Mar 07 10:15:00 myserver systemd[1]: nginx.service: Failed with result
```
```bash
# Useful commands:
sudo tail -50 /var/log/syslog                          # last 50 lines
sudo tail -f /var/log/syslog                           # follow live
sudo grep -i "error\|failed\|critical" /var/log/syslog # find problems
sudo grep "nginx" /var/log/syslog                      # filter by service
sudo grep "Mar 07 10:" /var/log/syslog                 # filter by time
```

### 2. auth.log / secure вЂ” The Security Register
```
Location (Debian/Ubuntu): /var/log/auth.log
Location (RedHat/CentOS):  /var/log/secure
Purpose: Every login, logout, failed attempt, sudo command, SSH event
```
```bash
# Sample content:
Mar 07 09:00:01 myserver sshd[1001]: Accepted publickey for rahul from 192.168.1.10
Mar 07 09:05:33 myserver sudo: rahul: COMMAND=/bin/apt update
Mar 07 09:17:45 myserver sshd[1099]: Failed password for root from 45.33.32.156
Mar 07 09:17:46 myserver sshd[1099]: Failed password for root from 45.33.32.156
Mar 07 09:17:47 myserver sshd[1099]: Failed password for root from 45.33.32.156
```
```
Reading this like a detective:
  Line 1: rahul logged in via SSH key вњ… Normal
  Line 2: rahul ran apt update with sudo вњ… Normal
  Lines 3-5: ROOT login attempted 3 times from external IP рџ”ґ ATTACK!
```
```bash
# Useful commands:
sudo grep "Accepted" /var/log/auth.log                 # successful logins
sudo grep "Failed password" /var/log/auth.log          # failed attempts
sudo grep "sudo" /var/log/auth.log                     # sudo usage
sudo grep "Failed password" /var/log/auth.log \
     | awk '{print $11}' | sort | uniq -c | sort -rn   # top attacking IPs
```

### 3. kern.log вЂ” The Kernel Notebook
```
Location: /var/log/kern.log
Purpose: Kernel messages, hardware events, driver issues, memory problems
```
```bash
# Sample content:
Mar 07 07:59:55 myserver kernel: [0.000000] Linux version 5.15.0 starting
Mar 07 10:22:14 myserver kernel: [9823.445] EXT4-fs error (device sda1)
Mar 07 10:22:15 myserver kernel: [9824.112] Out of memory: Kill process 4521
Mar 07 10:22:15 myserver kernel: [9824.113] Killed process 4521 (python3)
```
```
The [number] in brackets = seconds since system booted
  [9824.112] = 9824 seconds after boot = ~2.7 hours after startup
```
```bash
# Useful commands:
sudo tail -30 /var/log/kern.log
sudo grep -i "error\|fail\|warn" /var/log/kern.log     # kernel errors
sudo grep -i "sda\|sdb\|disk\|ext4" /var/log/kern.log  # disk errors
sudo grep -i "out of memory\|oom" /var/log/kern.log    # memory kills
```

### 4. dmesg вЂ” Boot Hardware Detective
```
Location: /var/log/dmesg  (also accessible via dmesg command)
Purpose: What happened during boot вЂ” hardware detection, driver loading
dmesg = "diagnostic messages" вЂ” kernel ring buffer
```
```bash
# Commands:
dmesg                          # print kernel ring buffer
dmesg -T                       # with human-readable timestamps в­ђ
dmesg -H                       # human readable + colors
dmesg -l err,warn              # show only errors and warnings в­ђ
dmesg | tail -20               # last 20 kernel messages
dmesg | grep -i "usb"          # USB device messages
dmesg | grep -i "eth\|network" # network messages
```

### 7. Binary Log Files вЂ” Special Commands Required
```
These files are NOT plain text вЂ” do NOT use cat on them!
Use the special commands listed below.
```

| File | Contents | Command |
|---|---|---|
| `/var/log/wtmp` | Complete login/logout history | `last` or `last -20` |
| `/var/log/btmp` | Failed login attempts | `sudo lastb` or `sudo lastb -20` |
| `/var/log/lastlog` | Last login time per user | `lastlog` |
| `/var/log/faillog` | Failed login count per user | `sudo faillog -a` |

```bash
# Examples:
last | head -20            # recent login/logout history
sudo lastb | head -20      # failed login attempts
lastlog | grep -v "Never"  # users who have logged in (skip never-logged)
```

## рџ—єпёЏ Problem в†’ Log File Map (The Cheat Sheet)

| Problem You're Investigating | First Place to Look | Second Place |
|---|---|---|
| Can't SSH / login issues | `/var/log/auth.log` | `sudo lastb` |
| Service crashed / failed | `journalctl -u <service>` | `/var/log/syslog` |
| Disk or hardware errors | `dmesg -T -l err,warn` | `/var/log/kern.log` |
| System won't boot | `/var/log/boot.log` | `dmesg` |
| Website returning errors | `/var/log/nginx/error.log` | `/var/log/nginx/access.log` |
| Database not starting | `/var/log/mysql/error.log` | `journalctl -u mysql` |
| Software update broke something | `/var/log/apt/history.log` | `/var/log/dpkg.log` |
| Scheduled task not running | `/var/log/cron.log` | `journalctl -u cron` |
| Possible hack / intrusion | `/var/log/auth.log` | `sudo lastb` |
| General "something is wrong" | `/var/log/syslog` | `journalctl -b -p err` |
| Who logged in recently | `last \| head -20` | `/var/log/auth.log` |
