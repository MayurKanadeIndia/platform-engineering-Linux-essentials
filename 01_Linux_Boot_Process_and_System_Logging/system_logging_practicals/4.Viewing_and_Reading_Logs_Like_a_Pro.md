# 📖 Chapter 4 — Viewing & Reading Logs Like a Pro
> Reference guide with concepts, analogies, commands & key takeaways

---

## 🎯 Analogy Summary

### The Library Analogy
```
A huge library = your log files (millions of lines)

❌ Beginner approach:
   Read every book from page 1 to find one sentence
   → Takes forever, impossible at scale

✅ Professional approach:
   Use the index, search by keyword, filter by date/section
   → Find exactly what you need in seconds

The tools in this chapter = your library index system.
tail, grep, journalctl are your search tools.
Pipes (|) connect them into powerful search engines.
```

---

## 💡 Core Concept

> Never read log files manually from top to bottom.
> Always **filter → search → pipe** to find exactly
> what you need. The right command finds the answer
> in seconds, even in files with millions of lines.

## 🛠️ Complete Tool Overview

| Tool | Purpose | Best For |
|---|---|---|
| `cat` | Print entire file | Small files only |
| `tail` | See last N lines / follow live | ⭐ Daily use — most common |
| `head` | See first N lines | Oldest entries |
| `less` | Interactive scrollable viewer | Manual browsing |
| `grep` | Search / filter by keyword | ⭐ Finding specific events |
| `awk` | Extract specific columns | Parsing log fields |
| `wc` | Count lines/words | Measuring event frequency |
| `sort` | Sort output | Ordering results |
| `uniq -c` | Count duplicates | Finding top offenders |
| `\|` (pipe) | Connect tools together | ⭐ Building power combos |
| `journalctl` | Read journald logs | ⭐ Modern system logs |

## 🔧 Tool 1 — cat

```bash
cat /var/log/syslog              # print entire file
cat -n /var/log/syslog           # with line numbers
cat /var/log/syslog /var/log/auth.log  # multiple files at once
```
> ⚠️ **Warning:** Never run plain `cat` on large log files.
> They can have millions of lines and will flood your terminal.
> Use `tail`, `grep`, or `less` instead for large files.

---

## 🔧 Tool 2 — tail ⭐ Most Used Log Command

```bash
# Basic usage
tail /var/log/syslog             # last 10 lines (default)
tail -n 50 /var/log/syslog       # last 50 lines
tail -100 /var/log/syslog        # last 100 lines

# ── FOLLOW MODE — The Most Powerful Option ──────────────────
tail -f /var/log/syslog          # follow live in real time ⭐
tail -f /var/log/auth.log        # follow auth events live
tail -f /var/log/nginx/access.log  # follow web traffic live

# Follow multiple files at the same time
tail -f /var/log/syslog /var/log/auth.log

# Follow + filter (the professional combo) ⭐
tail -f /var/log/syslog | grep -i "error"
tail -f /var/log/auth.log | grep "Failed"
tail -f /var/log/nginx/access.log | grep " 500 "
```
> Press `Ctrl + C` to stop following.

---

## 🔧 Tool 3 — head

```bash
head /var/log/dpkg.log           # first 10 lines (default)
head -n 20 /var/log/dpkg.log     # first 20 lines
head -5 /var/log/syslog          # first 5 lines
```
> Use head when you want the **oldest** entries in a file,
> or to check when logging started.

## 🔧 Tool 4 — less (Interactive Viewer)

```bash
sudo less /var/log/syslog        # open interactive viewer
sudo less +G /var/log/syslog     # open at BOTTOM (newest logs) ⭐
```

### Navigation Keys Inside less
| Key | Action |
|---|---|
| `↑` / `↓` | Scroll one line up/down |
| `Space` | Page down |
| `b` | Page up |
| `G` | Jump to END (newest entries) |
| `g` | Jump to START (oldest entries) |
| `/keyword` | Search forward for keyword |
| `?keyword` | Search backward for keyword |
| `n` | Next search result |
| `N` | Previous search result |
| `q` | **QUIT** less |

```
How to search inside less:
  1. Open:       sudo less /var/log/syslog
  2. Type:       /error          (forward slash + keyword)
  3. Press Enter → jumps to first match
  4. Press n     → next match
  5. Press N     → previous match
  6. Press q     → quit
```

## 🔧 Tool 5 — grep ⭐ Most Powerful Search Tool

### Basic grep
```bash
grep "error" /var/log/syslog                   # exact match, case sensitive
grep -i "error" /var/log/syslog                # case insensitive ⭐ always use this
grep -n "error" /var/log/syslog                # show line numbers
grep -v "info" /var/log/syslog                 # lines NOT matching (inverse)
grep -c "Failed" /var/log/auth.log             # count matching lines only
grep -r "error" /var/log/                      # search ALL files recursively
```

### grep with Multiple Keywords
```bash
# OR — match any of these keywords
grep -E "error|failed|critical" /var/log/syslog

# AND — must match BOTH (pipe two greps)
grep "ssh" /var/log/auth.log | grep "Failed"

# Search multiple files at once
grep "error" /var/log/syslog /var/log/auth.log
```

### grep with Context (Show Surrounding Lines) ⭐
```bash
grep -A 3 "error" /var/log/syslog    # 3 lines AFTER the match
grep -B 3 "error" /var/log/syslog    # 3 lines BEFORE the match
grep -C 3 "error" /var/log/syslog    # 3 lines BEFORE AND AFTER ⭐
```
```
Example output of grep -C 2 "Out of memory":

  Mar 07 10:22:12 kernel: Memory pressure increasing       ← 2 before
  Mar 07 10:22:13 kernel: Swap usage at 95%                ← 1 before
  Mar 07 10:22:14 kernel: Out of memory: Kill process 4521 ← MATCH
  Mar 07 10:22:14 kernel: Killed process 4521 (python3)    ← 1 after
  Mar 07 10:22:15 kernel: Memory freed: 450MB              ← 2 after

Now you see the full STORY around the error!
```

### Real-World grep Examples
```bash
# Failed SSH logins
sudo grep "Failed password" /var/log/auth.log

# Extract just the IP addresses from failed logins
sudo grep "Failed password" /var/log/auth.log | grep -oP 'from \K[\d.]+'

# Sudo commands run today
sudo grep "sudo" /var/log/auth.log | grep "Mar 07"

# Nginx 500 errors
sudo grep " 500 " /var/log/nginx/access.log

# Critical system messages
sudo grep -iE "critical|emergency|panic" /var/log/syslog

# Disk I/O errors
sudo grep -i "I/O error\|disk error\|EXT4-fs error" /var/log/kern.log
```

## 🔧 Tool 6 — Pipes ( | ) ⭐ Building Power Combos

```
What is a pipe?
  command1 | command2 | command3
      ↓           ↓          ↓
  produces    filters    further processes
  output      results    the results

Output of one command becomes input of the next.
```

### Power Combo Examples

```bash
# ── Top Attacking IPs ───────────────────────────────────────
sudo grep "Failed password" /var/log/auth.log \
     | awk '{print $11}' \
     | sort \
     | uniq -c \
     | sort -rn \
     | head -10

# Output:
#  500 45.33.32.156   ← this IP tried 500 times! block it
#  120 192.168.1.55
#   45 10.0.0.22

# ── Most Visited Pages on Website ───────────────────────────
sudo cat /var/log/nginx/access.log \
     | awk '{print $7}' \
     | sort \
     | uniq -c \
     | sort -rn \
     | head -10

# Output:
#  5234 /index.html
#  3421 /about
#    45 /admin       ← someone probing admin page!

# ── Count Errors in Syslog ──────────────────────────────────
sudo grep -i "error" /var/log/syslog | wc -l

# ── Save Errors to File for Analysis ────────────────────────
sudo grep -i "error" /var/log/syslog > ~/errors_today.txt
cat ~/errors_today.txt | wc -l
```

## 🔧 Tool 7 — journalctl ⭐ Modern Log Reading

### Basic Commands
```bash
journalctl                         # ALL logs, oldest first
journalctl -r                      # ALL logs, newest first ⭐
journalctl -f                      # follow live (like tail -f) ⭐
journalctl -n 50                   # last 50 lines
journalctl --no-pager              # don't open in less (just print)
```

### Filter by BOOT ⭐
```bash
journalctl -b                      # current boot only ⭐
journalctl -b -1                   # PREVIOUS boot (crash investigation!) ⭐
journalctl -b -2                   # 2 boots ago
journalctl --list-boots            # list all boots available
```
```
Output of --list-boots:
  -3  abc123def  Mon 2026-03-04 08:00:15  Mon 2026-03-04 18:30:00
  -2  def456abc  Tue 2026-03-05 08:00:10  Tue 2026-03-05 17:45:00
  -1  ghi789xyz  Wed 2026-03-06 08:00:05  Wed 2026-03-06 23:58:00
   0  jkl012mno  Thu 2026-03-07 08:00:01  still running
```

### Filter by SERVICE ⭐
```bash
journalctl -u nginx                # nginx logs only
journalctl -u ssh                  # SSH service logs
journalctl -u mysql                # MySQL logs
journalctl -u nginx -f             # follow nginx logs live ⭐
journalctl -u nginx -u mysql       # multiple services at once
journalctl -u nginx -b             # nginx logs this boot only
```

### Filter by TIME ⭐
```bash
journalctl --since "2026-03-07 10:00:00"
journalctl --until "2026-03-07 11:00:00"
journalctl --since "2026-03-07 10:00:00" --until "2026-03-07 10:30:00"
journalctl --since "1 hour ago"    # ⭐ most commonly used
journalctl --since "30 minutes ago"
journalctl --since yesterday
journalctl --since today
```

### Filter by SEVERITY ⭐
```bash
journalctl -p err                  # errors and above ⭐
journalctl -p warning              # warnings and above
journalctl -p crit                 # critical and above
journalctl -b -p err               # errors from current boot
journalctl -u nginx -p err         # nginx errors only

# Severity levels to filter by:
# emerg, alert, crit, err, warning, notice, info, debug
```

### Output Formats
```bash
journalctl -o short                # default readable format
journalctl -o verbose              # full detail per entry
journalctl -o json-pretty          # JSON (great for scripts)
journalctl -o cat                  # message text only
```

### Combining Filters ⭐
```bash
journalctl -b -u nginx -p err              # nginx errors this boot
journalctl -u ssh --since "1 hour ago"    # ssh logs last hour
journalctl -b -1 -p err | tail -30        # errors before last crash
journalctl -b | grep -i "error"           # search with grep
```

### Kernel Logs via journalctl
```bash
journalctl -k                      # kernel messages only
journalctl -k -b                   # kernel messages this boot
journalctl -k -p err               # kernel errors only
```

## 🧪 logger — Generate Test Log Entries

```bash
# Write a simple test message to syslog
logger "Hello! This is my first test log entry"

# Write with specific facility and severity
logger -p auth.warning "Test security warning"
logger -p kern.err "Test kernel error simulation"
logger -p user.info "Application started successfully"
logger -p cron.notice "Backup task completed"

# Verify your entry was written
sudo grep "first test log entry" /var/log/syslog
sudo grep "Test security warning" /var/log/auth.log
```

## 📊 Quick Reference — grep Options

| Option | Meaning | Example |
|---|---|---|
| `-i` | Case insensitive | `grep -i "error"` |
| `-v` | Invert (NOT matching) | `grep -v "info"` |
| `-c` | Count matches | `grep -c "Failed"` |
| `-n` | Show line numbers | `grep -n "error"` |
| `-r` | Recursive search | `grep -r "error" /var/log/` |
| `-A N` | N lines after match | `grep -A 3 "error"` |
| `-B N` | N lines before match | `grep -B 3 "error"` |
| `-C N` | N lines both sides | `grep -C 3 "error"` ⭐ |
| `-E` | Extended regex (OR) | `grep -E "err\|fail"` |
| `-oP` | Extract match only | `grep -oP 'from \K[\d.]+'` |

## 📊 Quick Reference — journalctl Filters

| What to Filter | Command |
|---|---|
| Current boot | `journalctl -b` |
| Previous boot | `journalctl -b -1` |

## ✅ Key Takeaways

```
✅ tail -f = your real-time window into the system ⭐
✅ grep -i = always use case-insensitive for safety
✅ grep -C 3 = see context around errors (root cause faster)
✅ Pipe | = connects tools to build powerful one-liners
✅ journalctl -b -1 = investigate logs from before a crash ⭐
✅ journalctl -u <service> = focus on one service at a time ⭐
✅ journalctl -p err = show only errors, cut out the noise
✅ logger = generate test entries to verify logging works
✅ Never read millions of log lines manually — always filter first
✅ Combine --since + -u + -p in journalctl for surgical filtering
```
